## Findings

| Severity | File:line | Issue | Fix |
|---|---:|---|---|
| **HIGH** | `apps/frontend/app/components/TerminalGrid.vue:6` | `selectedIds` controls both grid membership and command target. Filtering the grid to selected terminals can silently change “Send to Selected”, while `All` can still send to terminals not visible in the grid. This is confusing and risky. | Split state: e.g. `gridVisibleIds` / `gridFilterMode` separate from `commandMode` targets. Add explicit “Visible” command target only if desired. |
| **HIGH** | `apps/frontend/app/components/TerminalTile.vue:14` / `apps/frontend/app/composables/useTerminalView.ts:117` | Every grid tile mounts a full terminal view and sends resize events. With 20 tiny tiles, this resizes live PTYs to tiny dimensions and can disrupt TUIs/output wrapping. Hidden terminals can remain at grid-tile size after leaving grid. | Add a grid/preview mode that does not resize the PTY, or only sends resize for the active/focused tile. Restore sane dimensions when entering focus mode. |
| **MEDIUM** | `apps/frontend/app/components/TerminalTile.vue:23` | “Clicking a tile makes it active” is only true when clicking inside the xterm host via `useTerminalView`; clicking the tile header, name, padding, or action buttons does not activate the tile. | Add `@mousedown="terminals.setActive(terminal.id)"` to the tile root or header/body, with appropriate `.stop` handling if needed. |
| **MEDIUM** | `apps/frontend/app/components/TerminalGrid.vue:6` / `apps/frontend/app/components/CommandBar.vue:63` | Switching focus → grid can hide the currently active terminal if any selection exists and the active terminal is not selected. Result: no active-tile highlight, and the user may think the active terminal disappeared. | On entering grid, either include active in the grid filter, clear/ask about stale grid selection, or show a clear “active terminal hidden by selection” affordance. |
| **MEDIUM** | `apps/frontend/app/components/TerminalTile.vue:23` | Tiles are not keyboard-selectable containers. Keyboard users can tab to buttons, but cannot focus/select a tile as a tile, and focus does not update `activeId`. | Give tiles `tabindex="0"`, `role="button"` or a better interactive role, `@focusin`/`@keydown.enter.space` to set active, and a visible `:focus-visible` style. |
| **MEDIUM** | `apps/frontend/app/components/TerminalGrid.vue:28` / `apps/frontend/app/components/TerminalGrid.vue:62` | `repeat(ceil(sqrt(count)), minmax(0, 1fr))` allows very narrow tiles on small widths or large counts. Header buttons and xterm content become cramped before the grid scrolls meaningfully. | Enforce a practical minimum tile width, e.g. compute columns from container width or use `minmax(260px, 1fr)` with responsive column reduction. |
| **LOW** | `apps/frontend/app/components/TerminalGrid.vue:20` | Empty grid copy still says “Tiling all 0” and tells users to tick terminals in the sidebar, but there are none. Minor dead-end wording. | For `items.length === 0`, hide the grid-bar hint or replace it with create-focused empty-state copy. |
| **LOW** | `apps/frontend/app/components/CommandBar.vue:63` | Layout toggle buttons are icon-only with `title` but no `aria-label`; poor accessibility and unclear for keyboard/screen-reader users. | Add `aria-label="Focus view"` / `aria-label="Grid view"` and consider visible active state text or tooltip component. |

No critical issues found. The main product decision to resolve is separating “what I can see in grid” from “what command broadcast targets”; keeping one checkbox state for both is the biggest UX footgun.
